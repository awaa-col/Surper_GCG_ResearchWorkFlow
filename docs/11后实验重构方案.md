# 11 及以后实验重构方案

> 目标：删除旧的 `Exp11-13` 占位式推进链条，按照当前论文主线与现有结果，重建一套更严谨的后续实验序列。

## 一、为什么要重排

旧链条的问题不是方向完全错，而是**顺序不严谨**：

1. 还没统一标签和人工复核口径，就先做 late component 因果映射。
2. 还没做 seed / split 稳健性复验，就把单次结果往论文主结论上抬。
3. `Exp10` 只在自杀主题成立，却过早往“普遍的 topic-safe manifold”方向扩展。

当前更稳的论文主线只有三条：

1. `L17` 附近存在 refusal execution bottleneck。
2. `L18-L25` 更像 late safety manifold，而不是单一 persona 轴。
3. 当前最值得写进论文的机制链条是 `exec` 与后层 identity/style/safe-response 成分的分工，而不是更激进的攻击叙事。

所以 `11+` 必须重排为：**先修评测，再做稳健性，再做跨主题与机制探索。**

## 二、新的实验序列

### Exp11：统一标签与人工复核包

脚本：`poc/experiments/exp_11_review_pack.py`

目标：
- 把输出统一拆成：
  - `hard_refusal`
  - `soft_refusal`
  - `resource_redirect`
  - `safe_educational`
  - `unsafe_with_disclaimer`
  - `direct_unsafe`
- 导出统一 CSV / JSONL 供人工复核。

为什么必须先做：
- 没有这一步，后面所有“方向对应什么行为”的说法都不严谨。

### Exp12：核心结论稳健性复验

脚本：`poc/experiments/exp_12_core_claim_robustness.py`

目标：
- 用多 seed / 固定 split / CI 复验当前论文主结论。

必须覆盖：
- `L17` bottleneck 是否稳定。
- `exec_only` 的行为改变是否稳定。
- `exec + identity-like steering` 的增益是否稳定。

### Exp13：跨主题复制 `Exp10`

脚本：`poc/experiments/exp_13_cross_topic_safe_manifold.py`

目标：
- 在 `self_harm / explosives / fraud / hate_or_abuse` 上复现 same-topic contrast。
- 验证 late safe-response 成分是否只在单主题成立，还是可以跨主题复现。

严谨性要求：
- `train/dev/test` 分离。
- 不再只看单组 prompt 的同集投影差。

### Exp14：自然软拒答 vs induced directions 对齐

脚本：`poc/experiments/exp_14_natural_soft_refusal_alignment.py`

目标：
- 检验自然生成出来的 `soft_refusal / resource_redirect / unsafe_with_disclaimer` 残差，
  是否真的与 system-prompt / prefill 提取出的方向对齐。

这一步的价值：
- 决定 `Exp07/09/09b` 得到的 induced directions 到底是“真机制 proxy”，还是“仅仅可用的操作手柄”。

### Exp15：detect-like signal 全层 survey

脚本：`poc/experiments/exp_15_detect_survey.py`

目标：
- 直接补 `r_detect` 的自然层位与传播路径证据。

要回答：
- detect 最早在哪层稳定出现？
- 它与 `r_exec` 的关系是什么？
- `exec ablation` 后 detect 方向是否保持一致？

## 三、执行顺序

5. `Exp15`

只有这五步跑通后，才建议重新回到：

- token attack / loss 设计回接

以下旧脚本已不再作为后续主线：

- `exp_11_topic_safe_response.py`
- `exp_12_late_component_causal.py`
- `exp_13_human_review_pack.py`

原因不是它们完全无效，而是它们对应的是**旧的推进顺序**，不适合继续作为论文后半段的主执行链条。
