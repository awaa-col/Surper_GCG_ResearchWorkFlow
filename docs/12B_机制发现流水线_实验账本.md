# 12B 机制发现流水线与实验账本

## 目标

这份文档把 `Exp00-31` 重新整理成面向 `12B` 的机制发现主线。

原则只有四条：

1. `1B` 结论只能当历史参考，不能当前提。
2. 以现有 JSON 结果和统一报告为准，不用口号覆盖证据。
3. 数据有限时，先缩小候选空间，再追加样本。
4. 攻击实验不再承担“发现机制”的职责，只做最终验收。

## 结论先行

当前最准确的判断是：

- `Exp18` 不能继续默认 `L17` 是 `12B` 的主 gate。
- `Exp19` 不能继续固定 `L17/L23` 做 `12B` 主线联动。
- 现有 `gate_scan` / `large_model_attack` 更接近“1B 迁移验证流程”，不是“12B 机制重新发现流程”。
- 目前能原样进入 `12B` 准确主线的脚本很少，真正可以直接当起点的是评测校准层。

## 1B 的真实工作流

把旧实验按证据链压缩后，`1B` 的真实流程是：

1. 基线诊断
2. refusal direction 提取
3. 逐层扫描与跨层验证
4. 盲评，排除伪成功
5. detect / exec 解缠
6. late safe-response 发现
7. 候选主线量化
8. gate 与 late 的联动分析
9. 更细的 trace / family / token 探索

`12B` 应该参考的是这条“研究顺序”，而不是其中的层位结论。

## 12B 准确主线

### Stage 0: Eval Calibration

- 目标：统一标签口径和人工复核。
- 当前可用脚本：`Exp11`
- 输出：
  - `hard_refusal`
  - `soft_refusal`
  - `resource_redirect`
  - `safe_educational`
  - `unsafe_with_disclaimer`
  - `direct_unsafe`
  - `incoherent`

### Stage 1: Baseline Diagnosis

- 目标：先看 `12B` 的原始拒答和系统提示敏感性。
- 参考模板：`Exp00`
- 输出：
  - baseline refusal profile
  - prompt format sensitivity
  - obvious jailbreak seams

### Stage 2: Gate Discovery

- 目标：全层无先验地发现主 gate 候选。
- 参考模板：`Exp01`、`Exp01-scan`
- 要求：
  - 每层各自提方向
  - 每层各自打自己
  - 输出 refusal / risk / incoherent
- 输出：
  - top gate candidate layers
  - destructive layers
  - provisional cooperating layers

### Stage 3: Cross-Layer Refinement

- 目标：只围绕候选层做跨层迁移和 pair 验证。
- 参考模板：`Exp01b`、`Exp01-blind`
- 输出：
  - best single-layer candidate
  - best pair candidate
  - transfer matrix
  - “真实释放 vs 退化”人工复核结果

### Stage 4: Detect Discovery

- 目标：独立发现 detect 线，不再从旧 `L17` 故事推断。
- 参考模板：`Exp05`、`Exp15`
- 输出：
  - earliest stable detect layers
  - detect 与 gate 的关系
  - detect 是否先于 gate

### Stage 5: Late Safe-Response Discovery

- 目标：找 `12B` 的晚层安全组织区，而不是直接写 persona。
- 参考模板：`Exp13`、`Exp16`、`Exp17`
- 输出：
  - cross-topic late manifold 是否存在
  - refusal / empathy / redirect / unsafe-exec family 候选
  - 哪些 family 是稳定的

### Stage 6: Candidate Quantification

- 目标：量化候选 gate，而不是默认 `L17`。
- 参考模板：`Exp18`
- 输出：
  - AUROC
  - effect size
  - projection gap
  - candidate ranking

### Stage 7: Minimal Causal Closure

- 目标：把上游 gate 和下游 late 结构连成最小闭环。
- 参考模板：`Exp19`
- 输出：
  - baseline
  - gate only
  - coop only
  - gate + coop
  - 必要时 detect only
- 通过标准：
  - 能区分真实危险释放和模型退化
  - 能描述 gate -> late family -> output form 的最小因果链

### Stage 8: Robustness

- 目标：只对已发现的闭环做多 seed / 多 split 复验。
- 参考模板：`Exp12`

### Stage 9: Attack Acceptance

- 目标：最后再做攻击验收。
- 对应：`Exp38`、`Exp39`
- 备注：这一层不再负责发现机制。

## 最终口径

当前最准确的仓库内口径应写成：

> `1B` 的实验流程可以作为 `12B` 机制发现的研究模板，但 `L17/L23` 及其相关量化和联动脚本不能直接作为 `12B` 的机制前提。现阶段，`12B` 的准确流水线应先从评测校准开始，再对 `gate discovery -> detect discovery -> late safe-response discovery -> candidate quantification -> minimal causal closure` 逐段重构。
