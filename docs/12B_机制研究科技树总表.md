# 12B 机制研究科技树总表

## 使用原则

这份文档只继承 `1B` 的**研究思路与排查顺序**，不继承 `1B` 的**层位、方向、模块结论**。

因此：

- `L17` 不是 `12B` 的默认主 gate。
- `L23` 不是 `12B` 的默认晚层协同位点。
- `r_exec / r_detect / late manifold` 在 `12B` 上都必须重新发现、重新命名、重新量化。
- 某实验只有在其前置“研究结果”成立后，才应该解锁。

文档目标不是复述旧结论，而是回答一件事：

> 如果我们要按 `1B` 的探索思路，在 `12B` 上重新摸索，那么每个实验到底依赖什么前提，做完又能解锁什么后续实验？

## T0 评测口径校准

### 节点定义

- 目标：先统一“什么算硬拒答、软拒答、改写后仍危险、直接危险”。
- 对应实验编号：`Exp11`
- 当前状态：可直接开跑。

### 前置研究

- 无。

### 必须产出

- 一套统一标签：
  - `hard_refusal`
  - `soft_refusal`
  - `resource_redirect`
  - `safe_educational`
  - `unsafe_with_disclaimer`
  - `direct_unsafe`
  - `incoherent`
- 一套人工复核样本与审计口径。

### 解锁

- 解锁 `T1` 之后几乎所有节点。
- 如果这一步没完成，后面所有“方向控制了什么行为”都会失真。

## T1 基线诊断

### 节点定义

- 目标：先确认 `12B` 在不做任何机制假设时的基础安全形态。
- 对应实验编号：`Exp00`
- 当前状态：方法可复用，脚本口径需要按 `12B` 重写。

### 前置研究

- `T0` 完成。

### 需要回答

- `12B` 的 baseline refusal profile 是什么。
- prompt 包装、system prompt、格式变化会不会显著影响拒答。
- 是否存在明显的天然缝隙或异常退化模式。

### 必须产出

- baseline label summary
- prompt format sensitivity
- obvious jailbreak seams

### 解锁

- 解锁 `T2` 的数据分层与候选 prompt 设计。

---

## T2 候选 Gate 无先验发现

### 节点定义

- 目标：不假设 `L17` 存在，直接在全层寻找 `12B` 的主 gate 候选。
- 对应实验编号：`Exp01`、`Exp01-scan`
- 当前状态：必须重构，现有实现仍带 `L17` 先验。

### 前置研究

- `T0` 已能稳定打标签。
- `T1` 已确认 baseline 与异常输出边界。

### 需要回答

- 哪些层最像“打一下就会释放危险输出”的 gate 候选。
- 哪些层主要导致乱码或整体退化，而不是真正释放危险行为。
- 哪些层可能与主 gate 协同，但自身不是主 gate。

### 必须产出

- top gate candidate layers
- destructive layers
- provisional cooperating layers
- 每层的 `refusal / risk / incoherent` 变化

### 解锁

- 解锁 `T3`、`T7`。
- 如果这一层没有得到候选列表，后面所有“量化”都是伪精确。

---

## T3 候选 Gate 跨层精炼

### 节点定义

- 目标：只围绕 `T2` 找到的少数候选层做跨层迁移、配对和盲评。
- 对应实验编号：`Exp01b`、`Exp01-blind`
- 当前状态：必须重构，不能再默认 `L17/L23`。

### 前置研究

- `T2` 至少产出一组单层候选与协同候选。

### 需要回答

- 最强单层候选是谁。
- 最强双层或小组合候选是谁。
- 候选方向跨层迁移时是“真实机制一致”还是“只是在某层碰巧好用”。
- 输出变化是真实释放，还是纯退化/胡言乱语。

### 必须产出

- best single-layer candidate
- best pair candidate
- transfer matrix
- 人工盲评结论：真实危险释放 vs 模型退化

### 解锁

- 解锁 `T4`、`T5`、`T7`。

---

## T4 Detect-like 信号重新发现

### 节点定义

- 目标：独立寻找 `12B` 的 detect-like 信号，不能把 `1B` 的 `r_detect` 直接搬过来。
- 对应实验编号：`Exp05`、`Exp15`
- 当前状态：必须重构，现有 detect proxy 带旧主线前提。

### 前置研究

- `T3` 至少确认一个可信的 gate 候选。

### 需要回答

- detect-like 信号最早稳定出现在哪些层。
- 它是否早于主 gate。
- 它与 gate 的关系是上游、并行还是弱相关。
- 自然前向与干预条件下，它是否保持一致。

### 必须产出

- earliest stable detect layers
- detect-gate relation
- baseline vs intervention consistency

### 解锁

- 解锁 `T8`。
- 若 detect 线不清楚，因果闭环只能写成弱版本。

---

## T5 晚层 Safe-Response 区重新发现

### 节点定义

- 目标：重新发现 `12B` 的晚层安全响应区，而不是先命名成 persona 或直接套用 `L18-L25`。
- 对应实验编号：`Exp13`、`Exp16`
- 当前状态：必须重构。

### 前置研究

- `T3` 已确认可信 gate 候选。
- `T0` 标签已经足够区分软拒答、转介、安全教育、带免责声明的不安全输出。

### 需要回答

- `12B` 是否存在跨主题共享的晚层 safe-response 空间。
- 这个空间里有哪些稳定成分。
- 这些成分是 topic-specific、style-like，还是更接近共享安全组织结构。

### 必须产出

- late safe-response candidate band
- cross-topic consistency
- family-level candidate list

### 解锁

- 解锁 `T6`、`T8`、`T11`。

## T6 候选 Family / Feature 整理

### 节点定义

- 目标：在 `T5` 已确认存在晚层区后，再去拆里面的 family、dictionary、feature。
- 对应实验编号：`Exp16`、`Exp17`
- 当前状态：条件式复用，不能抢跑。

### 前置研究

- `T5` 已确认晚层区真实存在。

### 需要回答

- 可以稳定提取哪些 family。
- 哪些 family 在多主题下仍存在。
- feature probe 是否支持这些 family 不是纯表层措辞。

### 必须产出

- refusal / apology / redirect / safe-info 等候选 family
- stable feature list
- family-feature 对应关系

### 解锁

- 解锁 `T8` 的晚层因果验证。
- 解锁 `T11` 的精细拆解。

## T7 候选 Gate 量化

### 节点定义

- 目标：量化的是 `T2/T3` 发现的候选 gate，而不是预设 `L17`。
- 对应实验编号：`Exp18`
- 当前状态：必须重构，当前脚本默认 `L17`。

### 前置研究

- `T3` 已给出可信的单层/双层候选。

### 需要回答

- 哪个候选 gate 真正具备最强区分力。
- 哪些候选只是有效，但不够稳定。
- 哪些方向虽然可操作，但不代表自然机制主轴。

### 必须产出

- AUROC
- effect size
- projection gap
- candidate ranking

### 解锁

- 解锁 `T8`
- 为 `T9` 提供复验对象

## T8 最小因果闭环

### 节点定义

- 目标：把上游 gate、可选 detect、下游晚层 family 连成最小闭环。
- 对应实验编号：`Exp19`
- 当前状态：必须重构，当前实现写死了 `L17/L23` 故事。

### 前置研究

- `T4` 至少给出 detect-like 线索，或者明确 detect 暂不纳入主闭环。
- `T5` 已确认晚层 safe-response 区。
- `T7` 已完成候选 gate 量化排序。

### 需要回答

- 仅动 gate 会发生什么。
- 仅动晚层协同位点会发生什么。
- gate + late family 是否形成最小可解释闭环。
- detect 是否是必要前置，还是只提供补充解释。

### 必须产出

- baseline
- gate only
- late only
- gate + late
- detect only 或 detect + gate 的补充条件
- 对“真实危险释放 vs 全局退化”的明确区分

### 解锁

- 解锁 `T9`
- 解锁 `T10`
- 解锁 `T11`

## T9 稳健性复验

### 节点定义

- 目标：只复验已经闭环的结论，不在机制没站稳时扩大战线。
- 对应实验编号：`Exp12`
- 当前状态：应后置，不应抢在主线发现之前。

### 前置研究

- `T8` 已得到最小闭环。

### 需要回答

- 多 seed 是否稳定。
- 多 split / 多 topic / 多模板是否稳定。
- 关键 effect 的置信区间是否仍然成立。

### 必须产出

- seed stability
- split stability
- bootstrap / Wilson CI
- failure cases

### 解锁

- 解锁 `T10`
- 让论文或统一报告口径可写成强结论

## T10 攻击验收

### 节点定义

- 目标：攻击实验只做最终验收，不再承担“发现机制”的职责。
- 对应实验编号：`Exp38`、`Exp39`

### 前置研究

- `T8` 至少成立。
- 最好 `T9` 已完成。

### 需要回答

- 已发现的最小闭环是否足以支持更强攻击或绕过。
- 攻击成功时，到底命中了哪一段已知机制。

### 必须产出

- attack success with mechanism attribution
- failure / bypass taxonomy

### 解锁

- 不再解锁主线，只做验收与外推。

## 当前最准确的统一口径

> `1B` 的实验结果不能直接迁移到 `12B`，但 `1B` 的探索顺序仍可作为 `12B` 的研究模板。
